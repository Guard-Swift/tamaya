# コーディングルール

本プロジェクト「たまや (tamaya)」における、HTML、CSS、およびJavaScriptのコーディング規則を定義します。AIエージェントはコードを生成・編集する際、必ず以下のルールに従ってください。

## 1. 最優先ルール（ファイルの分離）

*   **CSSおよびJavaScriptの完全外部ファイル化**:
    *   HTMLファイル内へのスタイル（`<style>` タグやインラインの `style` 属性）およびスクリプト（`<script>` タグによる直接記述）の書き込みは**禁止**します。
    *   スタイルはすべて `.css` ファイル、ロジックはすべて `.js` ファイルに記述し、HTMLから読み込んでください。
*   **ディレクトリ構成ルール**:
    *   外部ファイルは、以下の構成に従って整理してください。
        ```text
        tamaya/
        ├── index.html          # メインのHTMLファイル
        ├── css/
        │   └── style.css       # メインのスタイルシート
        ├── js/
        │   ├── main.js         # メインのJavaScriptロジック
        │   └── firework.js     # 花火のCanvasアニメーション用ロジックなど
        └── assets/
            ├── images/         # 画像アセット
            └── sounds/         # 効果音などの音声アセット
        ```

## 2. HTMLコーディングルール

*   **セマンティックHTMLの採用**:
    *   `<div>` や `<span>` を多用せず、`header`, `main`, `footer`, `section`, `article`, `nav` などの意味のあるHTML5タグを適切に使用してください。
*   **IDとクラスの使い分け**:
    *   `id` 属性は、JavaScriptからの特定要素の操作やアンカーリンクとしてのみ使用し、スタイリングの対象としては指定しないでください。
    *   スタイリングはすべて `class` 属性に対して行います。
*   **アクセシビリティとSEOの考慮**:
    *   すべての `<img>` タグには、適切な `alt` 属性を指定してください。
    *   見出しタグ（`<h1>` 〜 `<h6>`）は、階層構造（ドキュメントのアウトライン）に従って正しく使用してください。

## 3. CSSコーディングルール

*   **CSSカスタムプロパティ（変数）の活用**:
    *   夜空の背景グラデーション、ネオンカラー、フォントファミリー、トランジション速度など、プロジェクト全体で共有するデザインシステム・トークンは、`:root` でCSS変数として定義して管理してください。
*   **モバイルファーストとレスポンシブデザイン**:
    *   スマートフォンでの表示（デフォルト）を最優先とし、メディアクエリは `@media (min-width: ...)`（モバイルファースト）の形式で記述してください。
*   **アニメーションのパフォーマンス制限**:
    *   簡単なトランジションやホバーエフェクトはCSS（`transition`, `transform`, `opacity`）で実装してください。
    *   `width` や `height`, `top`, `left` などのレイアウト再計算（リフロー）を誘発するプロパティのアニメーションは避け、GPUで加速される `transform` や `opacity` を使用してください。

## 4. JavaScriptコーディングルール

*   **HTML/CSSとの関心の分離**:
    *   JavaScriptから要素のスタイル（`element.style.color` など）を直接変更することは原則として避けてください。
    *   代わりに、CSS側で状態（例: `.is-active`, `.is-hidden`）に対するスタイルを定義し、JavaScriptからは `classList.add()` や `classList.remove()` などのクラス操作のみを行ってください。
*   **厳格な変数・定数宣言**:
    *   変数の宣言には `const` または `let` を使用し、`var` の使用は禁止します。また、再代入しない変数は必ず `const` を使用してください。
*   **モダン機能の使用**:
    *   ES6+の仕様（アロー関数、テンプレートリテラル、分割代入など）を積極的に活用してください。
