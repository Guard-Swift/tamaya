# Git操作およびコミットルール

## 1. コミットメッセージの言語
* コミットメッセージは原則として**日本語**で記述してください。

## 2. コミットメッセージのフォーマット
* 変更内容がわかりやすいよう、以下のプレフィックスを冒頭に付与してください。
  * `feat:` 新機能の追加
  * `fix:` バグや不具合の修正
  * `docs:` ドキュメントの追加・修正
  * `style:` コードの意味に影響を与えない変更（フォーマット、空白の調整など）
  * `refactor:` リファクタリング（機能追加やバグ修正を含まないコード整理）
  * `chore:` ビルドプロセスやツール、ライブラリの更新などその他雑務
  * *例: `docs: Gitルールをエージェント設定に追加`*

## 3. コミットの粒度
* 複数の変更を1つにまとめず、論理的な作業単位（機能ごと、バグ修正ごと）でこまめにコミットを行ってください。

## 4. AIエージェントによるGit操作の制限
* AIエージェントがコミットやプッシュ、ブランチ作成などのGit操作を行う場合は、**必ず事前にユーザーの承認を得る**か、指示があった場合のみ実行してください。勝手なコミットやプッシュは禁止します。

## 5. Gitコマンドの実行場所
* Gitコマンドを実行する際は、カレントディレクトリ（Cwd）を必ず `tamaya` ディレクトリ（`/Users/matsuzawahiroshi/Desktop/project/tamaya`）に設定してください。親ディレクトリなどの異なるディレクトリで実行すると、不要なファイル走査が発生し実行に非常に時間がかかるため厳禁とします。
