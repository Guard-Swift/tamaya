# プロダクト要求仕様書 (PRD) : たまや (tamaya)

## 1. プロダクト概要
**たまや (tamaya)** は、複数人でコメントと花火の色をリレー形式でつなぎ、最後に「打ち上げる」ことで、全員のコメントと指定した色の花火が夜空を彩る、協調型の花火共有Webアプリケーションです。

*   **コンセプト**: 「みんなの想い（コメント）と個性（色）をのせて、夜空に大輪の花を咲かせる」
*   **想定デバイス**: スマートフォンでの利用をデフォルト（最優先）として設計。その場で手軽にスマホを操作・受け渡ししながら体験できることを重視します。
*   **体験の核**: 
    1.  一人ひとりが自分の「コメント」と「花火の色」を入力し、次の人にバトンタッチする。
    2.  バトンを受け取った人がさらにコメントと色を追加していく。
    3.  最後の人が「打ち上げる！」ボタンを押すと、それまでに集まったすべてのコメントと色が、美しい花火のアニメーションとなって夜空（画面）に打ち上がる。

---

## 2. ユーザー体験（UX）フロー

1.  **トップ画面 / 入力画面**:
    *   シンプルな夜空の背景。
    *   「コメント」入力フォーム（例：「合格おめでとう！」「いつもありがとう！」など）。
    *   「花火の色」選択UI（直感的なカラーパレット、またはカラーピッカー）。
    *   「次の人につなぐ（保存）」ボタン。
    *   「夜空に打ち上げる！」ボタン（これまでのデータを元に演出を開始する）。

2.  **リレーの進行（データ蓄積）**:
    *   「次の人につなぐ」を押すと、入力したデータが一時的に保存され、入力フォームがクリアされる（または「〇人目の花火がセットされました」と表示され、次の人がすぐに入力できる状態になる）。

3.  **打ち上げ演出（メイン演出）**:
    *   「夜空に打ち上げる！」が押されると、画面が全画面の夜空モードに切り替わる。
    *   それまでに蓄積されたコメントが画面上に美しく表示（例：メッセージがフェードイン、または花火の尾を引いて上昇するような演出）される。
    *   メッセージに続いて、それぞれのユーザーが指定した色で大輪の花火がドーンと打ち上がる。
    *   すべての花火が打ち上がった後、余韻を残して「もう一度つくる（リセット）」ボタンが表示される。

---

## 3. 機能要件

### 3.1. フロントエンド（UI/UX）
*   **モバイルファーストUI（スマートフォンがデフォルト）**: スマートフォンでの表示や片手操作をデフォルトとしてUI/UXを最適化。レスポンシブデザインによりPCでも美しく動作し、操作しやすいUI。
*   **リッチなデザイン**:
    *   夜空をイメージした深みのあるグラデーション背景。
    *   ボタンや入力フォームには、ネオンエフェクトや微細なホバーアニメーションを適用し、プレミアムな質感を演出。
*   **花火アニメーション (Canvas/CSS)**:
    *   HTML5 Canvas を用いた、パーティクルによる本格的かつ滑らかな花火エフェクト。
    *   指定された色（RGB/HEX）に基づいた正確な火花の再現。
*   **コメント表示エフェクト**:
    *   ただテキストを表示するだけでなく、花火の打ち上げ音（または視覚的な上昇エフェクト）と同期して、メッセージが浮かび上がる演出。

### 3.2. ユーザー管理 & データ永続化（データベース利用）
*   **簡易ユーザー登録・ログイン**:
    *   ユーザーは「ユーザー名」と「簡易的なパスワード」のみでアカウント登録・ログインが可能。
    *   複雑な認証手順（メール確認など）は省き、手軽に登録・ログインできるシンプルな仕組み。
*   **データベース（DB）によるデータ永続化**:
    *   登録したユーザーアカウント情報、および各ユーザーが投稿した「コメントと花火の色」のデータは、サーバー側のデータベースに保存・永続化。
    *   これにより、異なる端末やブラウザからでも、同じ打ち上げ用データ（コメントと色）を共有・蓄積可能。

### 3.3. コンテンツモデレーション（Google API連携）
*   **不適切コメントの自動判定・フィルタリング**:
    *   投稿されたコメントに、嫌がらせ、ヘイトスピーチ、露骨な表現などのセンシティブな内容が含まれていないかを自動判定。
    *   Google の **Natural Language API (Moderate Text)** または **Perspective API** などのモデレーションAPIを利用し、投稿時にリアルタイムでコンテンツをチェック。
    *   不適切なコンテンツと判定された場合は、データベースへの保存をブロックし、ユーザーに表現の変更を促すエラーメッセージを表示。

---

## 4. 非機能要件 / 品質要件
*   **操作の心地よさ**: 入力やボタンクリック時に、小気味よいアニメーションやフィードバックを返す。
*   **アニメーションの軽量さ**: スマートフォンでもカクつかずに60fpsに近い滑らかさで花火が描画されること。

---

## 5. 今後の拡張アイデア (Optional)
*   **ルーム機能（オンライン対応）**: FirebaseやWebSocket等を利用し、離れた場所にいる人同士が同じ「部屋」に入り、リアルタイムにコメントと色を投稿し合えるようにする。
*   **BGM/効果音**: 打ち上げ時の「ヒュ〜〜〜、ドーン！」という効果音や、情緒ある夜の環境音の追加。
